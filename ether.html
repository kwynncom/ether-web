the minting "data" param

First, note that if the code is verified as noted in the tutorial and with the command above, you can mint on EtherScan from the contract, such as my second test contract that I used to any degree. Put 0.08 as the payable amount and whatever address you want to receive the token in the other param.

If you do it that way, you can see the "data" param in MetaMask. That data param is the ABI / application binary interface. In this case, it's the "mintTo" function's signature--params, types, etc.--in Ethereum Virtual Machine (EVM) bytecode / binary code. You need that param in the client-side (web) JavaScript, as shown in my example.

I'm sure there are other ways to derive the param, and I've seen reference to them, but that works for now. Hopefully I will update this when I find a way that I can use before the contract is deployed.
